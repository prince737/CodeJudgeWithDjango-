Instructions / Rules

  • assignmentGeneral
    • Maximum of 2 participants is allowed in each Team.
    • The event will consist of 2 rounds - Prelims and Finale. Each round will be for 2 hours.
    • Top 10 teams from the Prelims will be selected for the finale.
    • The winning and 1st Runner up team will be rewarded.
    • Both round will contain 5 pure programming problems.
    • Each problem will carry 20 marks.
    • On submission you will either get Full Marks or none at all depending on the result. (Refer to leaderboard section to find out how ranking works)
    • On each submission that is not accepted as correct, +5mins penalty will be added to your total time.
    • Participants can use any language from C, C++, Java, Python 3.
    • Each team can use only one computer.
    • Internet and Book access during the contest is strictly prohibited. If any participant or team is found indulging in unfair means then that participant or team will be disqualified immediately.
    • For C/C++, gcc/g++ compiler will be used, for Java, jdk 1.8 will be used, for Python latest version (of 3.x) will be used.
    • External IDE's can be used.
  • codeRunning your Code
    • Below the provided code editor you will find a button "Run Code". Simply clicking that button will run the code currently present in the editor but make sure you choose the correct language from the dropdown.
    • By default "Run Code" assumes no input is needed for your code.
    • If you want to test your code against custom input, click on the checkbox named "Run against custom input" at the bottom left of the editor. This will open a text areafor you to provide proper input. Then simply click on "Run Code button to test against current input."
  • sendSubmitting your Code
    • To submit your code click on "Submit Code" button present at the bottom right corner of the editor.
    • It is important that you are on the same question tab which you wish to submit. Being on a certain question tab and submitting for some other question will result in your answer being treated as wrong.
    • Upon submitting your code will be tested against hidden test cases.
    • After you submit 4 cases can happen.
      1. Accepted: This means your code successfully passed all the test cases and 20 points will be added to your total score.
      2. Wrong Answer: This means the output from your code did not match the expected output. 5mins penalty will be added to your total time.
      3. Compilation Error: As the name suggests, this means you code compiled with error. Again 5 mins penalty will be added.
      4. Time Limit Exceeded: This means your algorithm took way too much time and you need to improve it. 5 mins penalty will be added.
  • assessmentLeaderboard
    • Teams will be placed on the leaderboard in descending order of total points earned by them.
    • Tie breaking rules: If two teams have equal total points on the leaderboard then the team with the lowest Total Time taken to solve the problems (including the added penalties) will be placed higher than the other.
    • The above scoring and ranking system is applicable for all the rounds of this competition.
